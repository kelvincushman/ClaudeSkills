# Atopile Publishing Reference

## `ato.yaml` Configuration

Before publishing, ensure your `ato.yaml` includes the following fields:

```yaml
name: your-package-name
version: 0.1.0
description: A brief description of your package.
license: MIT
```

## Best Practices

*   **Semantic Versioning**: Follow semver (e.g., 0.1.0, 1.0.0).
*   **Documentation**: Include a clear `README.md` in your package.
*   **Dependencies**: Ensure all dependencies are publicly available.
*   **Testing**: Build your project one last time before publishing to ensure it compiles correctly.
